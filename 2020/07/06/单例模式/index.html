    1.单例模式

    定义： 保证一个类只有一个实例，且该类只能通过一种方式创建这个实例。

    三个特点：

    • 只能创建一个实例对象，减少了内存的开销
    • 该单例对象必须由单例类自己创建
    • 设置全局访问点，严格控制访问

    一些例子： Windows 的回收站、操作系统中的文件系统、多线程中的线程池、显卡的驱动程序对象、打印机的后台处理服务、应用程序的日志对象、数据库的连接池、网站的计数器、Web 应用的配置对象、应用程序中的对话框、系统中的缓存等常常被设计成单例。

    学习单例的重点内容：

    • 私有构造器
    • 线程安全
    • 延迟加载
    • 序列化和反序列化安全
    • 反射

    第一种,懒汉式单例

    顾名思义，懒汉式单例比较懒，他只有在调用getInstance时才会创建实例，类加载时并不创建实例对象。

    代码实现：

    public class LazySingleton {

private volatile static  LazySingleton lazySingleton = null;

private LazySingleton(){

}

public synchronized static  LazySingleton getInstance(){
    if (lazySingleton == null) {
        lazySingleton = new LazySingleton();
    }

    return lazySingleton;

    }

}
    **懒汉式单例需要注意线程安全问题，当多线程运行时，如果不加锁，会造成创建多个实例。**

    使用doublecheck的当时显示懒汉式单例：

    public class LazyDoubleCheckSingleton {

private volatile static LazyDoubleCheckSingleton instance = null;

private LazyDoubleCheckSingleton(){

}

public  static LazyDoubleCheckSingleton getInstance(){


    if (instance == null) {
        synchronized(LazyDoubleCheckSingleton.class){
            if (instance == null){
                instance = new LazyDoubleCheckSingleton();
                //1.分配内存
                //2.初始化对象
                //3.设置对象指向分配的内存
            }
        }

    }

    return instance;

}

}
    **instance = new LazyDoubleCheckSingleton();这句话会分为三步进行，可能会造成指令的重排序，也就是会按照1.3.2去执行。在第一个进程执行完1.3步后，第二个进程会进来访问对象，此时对象并没有初始化，程序将会报错。** 原理：在加了volatile后，所有线程都能看到共享内存的工作状态，保存了内存的可见性。用volatile修饰的共享变量，在进行写操作的时候多会多出很多汇编代码，起到两个作用：第一，将当前处理器缓存的数据写回到系统内存；这个操作会使其他cpu的缓存无效，他们会从共享内存同步数据，从而保证了内存的可见性。这里用到了缓存一致性协议。

    使用功能静态内部类的方式实现懒加载

    代码实现
    public class StaticInnerClassSingleton {

        private StaticInnerClassSingleton(){

    }


    private static class InnerClass{
        private static StaticInnerClassSingleton instance = new StaticInnerClassSingleton();
    }

    public static  StaticInnerClassSingleton getInstance(){
        return  InnerClass.instance;
    }


}

    原理

    关键点在于在类的初始化过程中JVM会初始化一个锁即初始化对象锁，因而使用静态内部类使得多线程互不影响

    第二种，饿汉式单例

    代码实现：

    public class HungrySingleton {

private  final  static  HungrySingleton instance;

static {
    instance = new HungrySingleton();
}

public static HungrySingleton getInstance(){
    return  instance;
}



}

    顾名思义，在类加载时就已经实现了实例的初始化。

    序列化与反序列化将会破坏单例模式，解决方法：在Singleton类中加一个如下方法

     private Object readResovle(){
    return  instance;
}

    反射攻击破坏单例模式

    我们用反射来构造实例，从而破坏单例模式。
    饿汉式单例的解决方法：在私有构造器中加入判断

     private HungrySingleton(){
    if (instance !=null) {
        throw  new RuntimeException("休想反射");
    }
}

    对于懒汉式暂时无法防御反射攻击。

    通过枚举实现单例模式，这种方式也是Effective Java 这本书推荐的方式

    代码如下：

    public enum  EnumInstance {
INSTANCE;

private  Object Data;

public Object getData() {
    return Data;
}

public void setData(Object data) {
    Data = data;
}

public static  EnumInstance getInstance(){
    return  INSTANCE;
}
}

    使用这种方式可以防止序列化与反序列化，以及反射攻击。我们来看看它是如何做到的：

    在读取对象的时候，通过查看OjectInputStream源码，可以得出Enum是通过

    en = Enum.valueOf(cl,name);

    进行的Enum的初始化，因为枚举中的name是唯一的，并且对应一个枚举常量，所以拿到的常量也是唯一的。

    下图是Constructor的源码，在通过反射创建实例时，若发现类的类型为枚举则会报错，因此不能通过反射来创建一个新的实例。

    使用容器实现单例模式

    代码如下

    public class ContainerSingleton {

private static  Map<String,Object> singletonMap = new HashMap<String, Object>();

private static void putInstance(String key, Object instance){
    if (StringUtils.isNotBlank(key) &&  instance != null) {
        if (!singletonMap.containsKey(key)){
            singletonMap.put(key,instance);
        }
    }

}

public static Object getInstance(String key){
    return singletonMap.get(key);
}

    }
